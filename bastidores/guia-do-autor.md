# Guia do Autor

Este documento reúne os princípios utilizados para escrever os conteúdos do Linux Explorer.

## Público-alvo

Iniciantes.

Nunca assumir conhecimento prévio.

Sempre explicar novos termos na primeira vez que aparecerem.

---

## Linguagem

Utilizar linguagem simples.

Evitar jargões desnecessários.

Explicar conceitos antes de apresentar comandos.

---

## Objetivo dos capítulos

Cada capítulo deve responder apenas uma pergunta principal.

Exemplos:

* O que é Linux?
* O que é o Kernel?
* O que é uma distribuição?
* O que é o Terminal?

---

## Evolução do conteúdo

Cada capítulo deve preparar naturalmente o leitor para o próximo.

Nunca antecipar conteúdos que ainda serão estudados em detalhes.

---

## Qualidade

Antes de publicar um capítulo, verificar:

* Clareza.
* Organização.
* Exemplos.
* Laboratório.
* Erros comuns.
* Referências.
