# Checklist de qualidade

Antes de publicar um capítulo, confirme:

* [ ] O título está claro.
* [ ] A introdução desperta interesse.
* [ ] Os objetivos estão definidos.
* [ ] Os conceitos foram explicados antes da prática.
* [ ] Os exemplos foram testados.
* [ ] Existe uma seção "Pratique".
* [ ] Existe uma seção "Explore".
* [ ] Há um resumo rápido (quando aplicável).
* [ ] Os erros comuns foram apresentados.
* [ ] As referências são oficiais.
* [ ] A conclusão conecta o capítulo ao próximo.

# Filosofia do Linux Explorer

Todo capítulo deve seguir os seguintes princípios:

* Explicar antes de executar.
* Ensinar conceitos antes de comandos.
* Evoluir do simples para o complexo.
* Utilizar exemplos reais.
* Incentivar a prática.
* Desenvolver autonomia.
* Formar profissionais capazes de compreender o sistema, e não apenas decorar comandos.
