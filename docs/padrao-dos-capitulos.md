# Padrão dos Capítulos

Este documento define o padrão editorial usado nos conteúdos do Linux Explorer.

## Estrutura padrão

Cada capítulo deve seguir, sempre que fizer sentido, esta estrutura:

1. Visão Geral
2. Objetivos
3. Por que isso é importante?
4. Conceitos
5. Como funciona
6. Exemplo prático
7. Laboratório
8. Erros comuns
9. Boas práticas
10. Curiosidade
11. Referências
12. Conclusão
13. Próximo capítulo

## Regras editoriais

- Escrever para iniciantes.
- Não assumir conhecimento prévio.
- Explicar termos técnicos na primeira vez que aparecerem.
- Não introduzir muitos conceitos novos no mesmo capítulo.
- Cada capítulo deve responder uma pergunta principal.
- Evitar comandos sem explicar o objetivo.
- Evitar conteúdo desnecessariamente avançado.
- Usar linguagem clara, direta e progressiva.

## Estilo

- Títulos claros.
- Exemplos simples.
- Analogias quando ajudarem.
- Laboratórios curtos e práticos.
- Erros comuns baseados em dificuldades reais.
